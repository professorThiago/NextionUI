◆ nexEnviarCmd()

void nexEnviarCmd ( const char *  cmd)

Envia um comando ao Nextion seguido do terminador 0xFF 0xFF 0xFF.

Descarta qualquer dado pendente no buffer de entrada antes de enviar, garantindo que a resposta seguinte pertença a este comando.

Parâmetros
cmdString de comando terminada em \0.
Exemplo
nexEnviarCmd("page p1_dash");
nexEnviarCmd("t0.txt=\"Olá\"");

◆ nexIniciar()

bool nexIniciar ( uint32_t  baudrate = 9600,
int8_t  pinoRX = -1,
int8_t  pinoTX = -1,
HardwareSerial *  serial = &Serial1 
)

Inicializa a comunicação serial com o display Nextion.

Configura a porta serial, envia comandos iniciais (bkcmd=0 para suprimir retornos desnecessários) e aguarda o display estabilizar.

Parâmetros
baudrateVelocidade da comunicação. Padrão: 9600. Recomendado: 115200 para melhor desempenho.
pinoRXPino RX do ESP32 (recebe dados do Nextion TX). Use -1 para o padrão da porta serial escolhida.
pinoTXPino TX do ESP32 (envia dados para o Nextion RX). Use -1 para o padrão da porta serial escolhida.
serialPonteiro para a porta serial a usar. Padrão: &Serial1.
Retorna
true se inicializado com sucesso.
false se falhou (porta serial inválida).
Exemplo (ESP32 com pinos customizados)
// GPIO16 = RX do ESP32 (liga no TX do Nextion)
// GPIO17 = TX do ESP32 (liga no RX do Nextion)
nexIniciar(115200, 16, 17, &Serial2);

◆ nexLerNumero()

bool nexLerNumero ( uint32_t *  numero,
uint32_t  timeout = 100 
)

Lê um número (uint32_t) enviado pelo Nextion.

Formato esperado do Nextion: 0x71 <b0> <b1> <b2> <b3> 0xFF 0xFF 0xFF (little-endian, 4 bytes).

Parâmetros
[out]numeroPonteiro onde o número lido será armazenado.
timeoutTimeout em ms.
Retorna
true se lido com sucesso.
false se timeout ou formato inválido.
Exemplo
uint32_t val;
nexEnviarCmd("get n0.val");
if (nexLerNumero(&val)) {
Serial.println(val);
}
